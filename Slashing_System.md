# Slashing System（罰沒機制）


## 1. 設計原則

參考 Hyperliquid HIP-3 的罰沒機制，並結合 Orderly 的業務特點設計：

| 原則 | 說明 |
|------|------|
| 透明性 | 所有罰沒條件必須明確定義，項目方在上架前需充分知悉 |
| 客觀優先 | 盡量以客觀可驗證的條件自動觸發，減少人為判斷空間 |
| 技術中立 | 罰沒不區分惡意行為或技術失誤，僅以結果為依據 |
| 申訴保障 | 所有罰沒均提供申訴期，確保項目方有申辯機會 |
| 補償隔離 | 罰沒的質押將銷毀或進入平台 IF，不直接分配給受影響用戶 |

---

## 2. 罰沒觸發條件

### 2.1 欺詐或虧損（100% 罰沒）

| 違規類型 | 說明 | 罰沒比例 |
|---------|------|---------|
| 詐騙項目 | 經證實為 Rug Pull 或詐騙項目 | 100% |
| 平台虧損 | 穿倉不補或其他因素導致平台虧損 | 100% |

### 2.2 營運違約（50% 罰沒）

| 違規類型 | 說明 | 罰沒比例 |
|---------|------|---------|
| 棄置 Symbol | 上架後完全放棄做市與維護，導致流動性枯竭 | 50% |

### 2.3 市場操縱（0%-100% 罰沒，依嚴重程度）

| 違規類型 | 說明 | 罰沒比例 |
|---------|------|---------|
| 對敲交易 | 項目方錢包與自家 MM 對沖交易以虛增交易量 | 依嚴重性 0-50% |
| 價格操縱 | 透過大額交易刻意影響 Mark Price | 依嚴重性 0-50% |
| 清算獵殺 | 刻意觸發用戶清算以獲利 | 依嚴重性 50-100% |

**嚴重性評估因子**（由風控團隊評估）：
- 涉及金額佔總交易量比例
- 持續時間
- 受影響用戶數量
- 是否造成用戶實際損失
- 是否為首次違規


---

## 3. 罰沒決策流程

### 3.1 自動罰沒（客觀條件）

以下條件由系統自動執行，無需人工介入：

| 觸發條件 | 罰沒比例 | 執行方式 |
|---------|---------|---------|
| 項目方未補足 IF 超過 x 天 | 凍結相應比例 | 自動執行 |

### 3.2 人工審核罰沒

需要人工判斷的情況由 Orderly 風控團隊審核：

```
異常事件被檢測或舉報
  │
  ▼
Orderly 風控團隊調查（3-7 天）
  │
  ▼
發布調查結論與罰沒決定
  │
  ▼
通知項目方，提供 7 天申訴期
  │
  ├─── 無申訴或申訴駁回 ──► 執行罰沒
  │
  └─── 申訴成立 ──► 調整或取消罰沒
```

### 3.3 申訴機制

| 項目 | 說明 |
|------|------|
| 申訴期限 | 自動罰沒 48 小時；人工罰沒 7 天 |
| 申訴方式 | 提交書面說明與證據至指定郵箱 |
| 審核時間 | 5 個工作日內回覆 |
| 最終決定 | Orderly 風控團隊具有最終決定權 |

## 5. 執行階段建議

| Phase | 執行範圍 | 說明 |
|-------|---------|------|
| Phase 1 | 自動罰沒 | 合約地址不符、供應量造假、IF 未補足 |
